# v0.2.0 LAN Distribution вҖ” Architecture Design

**Date:** 2026-07-21
**Status:** Approved вҖ” ready for implementation plan
**Scope:** Repackage MetalFlow ERP as a double-click-installable desktop app for Windows, where one PC acts as server (DB + Next.js + UDP discovery) and 2-5 client PCs connect over LAN via Tauri 2.x shells.

## Problem

MetalFlow ERP runs today as a hosted web app (Postgres on remote server). The user wants to deploy it on-premise at a small workshop (2-5 PCs in one LAN) with:

1. **Installable with double-click** on Windows вҖ” no terminal, no Node.js, no manual Postgres setup.
2. **One PC acts as server**, the rest connect to it over LAN as clients.
3. **Local-first** вҖ” DB and app live on the server PC's filesystem, accessible from LAN.
4. **Cross-version safe** вҖ” clients detect version mismatch before opening the webview.

## Why pivot

Previous design (2026-07-05 spec) used Tauri-bundled-server: Tauri shell spawned Node child process. After 2+ hours of complex Rust lifecycle work (`tauri::generate_context!()` macro can only be invoked once per crate, requiring per-role binaries; ~2.4GB per worktree of Rust target/ artifacts), the user requested:

> "podriamos hacer una solucion mejor con un server node js y los clientes con tauri o electron"

This design **eliminates Tauri on the server side entirely**. The server is a pure Node PKG bundle; Tauri 2.x is used only for the thin client shells.

## Goal

Ship two products:

1. **`metalu-server.exe`** (~80MB, PKG bundle with Node 20 embedded + Next.js standalone + PGlite). Doble-click arranca, listens on `0.0.0.0:3000`, broadcasts UDP discovery on `:3001`, owns DB file in `%APPDATA%/metalu/db/`.
2. **`metalu-client.exe`** (~8MB, Tauri 2.x shell). On launch, listens for one server UDP broadcast, opens WebView2 window pointing at `http://<server-ip>:3000`, caches URL for 30 days.

All existing app code (modules, services, APIs, validation, auth) continues to work without modification. PGlite speaks real Postgres SQL, so Prisma schema is **unchanged**.

## Non-Goals (YAGNI вҖ” explicit out-of-scope for v1)

- HTTPS / TLS on LAN (LAN assumed trusted).
- Auto-update (Fran distributes new `.exe` manually via Dropbox/USB).
- Hot failover (if server PC dies, replace manually with backup).
- Scheduled/automated backups (button-triggered only).
- Code signing + Apple notarization (deferred until outside-the-taller distribution).
- Multi-write Postgres / multi-node concurrency (PGlite single-writer вҖ” fine for 2-5 PCs).
- Multi-platform: **Windows x64 only** (Mac/Linux deferred).
- Hot-reload during dev (manual restart).

**Key modules:**

- **`src/lib/discovery/announce.ts`** вҖ” `dgram.createSocket('udp4')`, `setInterval(broadcast, 5000)`. Reads local IP via `os.networkInterfaces()` (prefers IPv4 non-internal). Payload JSON validated against schema.
- **`src/server/entry.ts`** вҖ” PKG entrypoint: `require('next')()`, arranca en :3000, monta announcer, registra SIGINT handler.
- **`src/lib/backup/export.ts`** вҖ” `pglite.export()` returns Uint8Array, writes to `%APPDATA%/metalu/backups/metalu-YYYY-MM-DD.pglitebackup`.

**Por quГ© `src-tauri-client/` separado (no `src-tauri/`):**
- Server ya no es Tauri вҶ’ no hay dual build
- Un Гәnico binario cliente, una sola config
- CI matrix mГЎs simple (1 job Win x64 vs 4 del spec viejo)

**Flujo del cliente (`src-tauri-client/src/main.rs`):**
```rust
fn main() {
    let cached = server_url::read_cache();
    let url = match cached {
        Some(u) if u.expires_at > now() => u.url,
        _ => discovery::wait_for_server(30_000),
    };
    server_url::write_cache(&url);
    tauri::Builder::default()
        .setup(|app| {
            WebviewWindowBuilder::new(app, "main", url)
                .title("Metalu Cliente")
                .inner_size(1280.0, 800.0)
                .build()?;
            Ok(())
        })
        .run(tauri::generate_context!())?;
}
```

## Data flow

> Note: timings below are illustrative for Windows x64 with default disk/memory. Actual timings depend on hardware. Discovery latency (5s interval) is the only fixed timing contract.

### Server boot
```
[0ms]    PKG binary arranca Node embebido
[100ms]  src/server/entry.ts ejecuta require('next')
[500ms]  Next.js compila rutas on-demand
[1s]     src/lib/prisma/pglite.ts abre %APPDATA%/metalu/db/

[7.5s+]  broadcasts cada 5s
```

## Error handling

### Server-side

| Error | Manejo | UX |
|---|---|---|
| Puerto 3000 ocupado | PKG prueba 3001-3010, usa primero libre. Anuncia puerto real. | Transparente |
| PGlite write conflict (>5s timeout) | Service mapea a HTTP 503 + `Retry-After: 1` | Toast "Reintentando..." con backoff |
| PGlite corruption on startup | `pglite.open()` throws вҶ’ log error + sugerir restore | Banner en /admin |
| Migration failure | Server sigue arriba, loggea error. Backup panel: "DB inconsistente" | Banner persistente |
| Disk full (no se puede escribir backup) | API devuelve 500 | Toast: "Disco lleno" |

### Client-side

| Error | Manejo | UX |
|---|---|---|
| Discovery timeout (>30s sin broadcast) | Tauri main muestra modal | Modal: "No se encontrГі servidor. VerificГЎ que estГ© encendido y en la misma red WiFi." + botГіn "Reintentar" |
| Cache corrupta | Se descarta, vuelve a discovery | Transparente |
| HTTP 500 sostenido | Webview muestra error de Next.js con retry | Default Next.js error UI |
| HTTP 401 (token expirado) | next-auth redirige a /login | Transparente |

### Network

| Escenario | Manejo |
|---|---|
| LAN partition | Cliente no recibe UDP вҶ’ discovery timeout вҶ’ modal de error |
| Cliente pierde WiFi 10s | Webview fetch falla вҶ’ toast "Conectando..." + retry cada 5s |
| DHCP renueva IP server | Cliente detecta por failure de fetch. Re-discovery al reiniciar app. |

### Data integrity

| Escenario | Manejo |
|---|---|
| Backup file corrupto | `pglite.import()` throws вҶ’ 500. Backup NO se borra. |
| Restore parcial | PGlite rollback automГЎtico. UI: "Restore fallГі, DB sin cambios" |
| Dos restores simultГЎneos | Lock file `%APPDATA%/metalu/restore.lock`. Segundo: 409 Conflict |

## Migration from old spec

| Old spec (2026-07-05) | New design |
|---|---|
| Server = Tauri shell + spawn Node child | Server = Node PKG bundle, no Tauri |
| `src-tauri/` with dual build configs | `src-tauri-client/` single client config |
| `BUILD_TARGET=server\|client` env var | Eliminated вҖ” server is separate product |

| Tauri Rust UDP discovery (client side) | Same вҖ” Rust listener (unchanged) |
| `tauri::generate_context!()` macro per-role | Once per binary (only client) |
| ~21GB Rust toolchain | Eliminated (server) вҖ” only client builds Rust |

## Open decisions for implementation plan

- **PKG vs nexe vs SEA**: PKG (`@yao-pkg/pkg`) chosen вҖ” most mature, handles WASM via `--public-files`. SEA (Node 21+) considered but requires Node 21+ runtime.
- **Cache TTL**: 30 days. Configurable via env var `METALU_CLIENT_CACHE_DAYS` (default 30).
- **Backup retention**: keep last 10 backups, auto-prune older.
- **Restore lock TTL**: 5 minutes max (force-release if stale).
- **UDP port**: 3001 fixed (configurable via env var `METALU_DISCOVERY_PORT`).
- **First-user bootstrap**: seed creates `admin/admin` on first run. Subsequent users via admin UI (existing flow).

## Risks

- **`pkg` is community-maintained** вҖ” official Node SEA is the future direction. Mitigation: document migration path in README.
- **PGlite WASM inside PKG bundle** вҖ” must extract to temp dir at runtime (PKG `--public-files` handles this).
- **WebView2 missing on Windows 10** вҖ” Tauri shows error on launch. Mitigation: detect + show modal with WebView2 installer link.
- **Single-writer ceiling** вҖ” PGlite serializes; 6+ concurrent writes/sec may timeout. Acceptable for 2-5 PCs LAN.
- **`app_data_dir` ownership** вҖ” if `%APPDATA%/metalu/` is cloud-synced (OneDrive), PGlite file conflicts. Documented warning in /admin.
- **No TLS** вҖ” LAN assumed trusted. If workshop has guests on WiFi, document WPA2 isolation requirement.
